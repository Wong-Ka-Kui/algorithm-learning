# git-github

一键执行本地git提交和上传GitHub：检查git状态 → 更新项目README → 人工确认 → git提交推送

## 工作流程

### 阶段一：准备和确认

**1. 检查git状态**
- 运行 `git status` 查看所有变更文件
- 列出所有修改和新增的文件

**2. 分析今日学习内容**
- 查看修改的文件，识别今日新增的学习内容
- 重点关注：
  - `01-入门/1.2-常见的排序算法/代码/` 下的新增Java文件
  - `01-入门/1.2-常见的排序算法/算法题/` 下的新增题目
  - `01-入门/1.2-常见的排序算法/README.md` 的更新内容
  - 其他子模块的新增内容

**3. 更新项目根目录README**
- 读取 `README.md`
- 在「已完成内容」章节中补充今日新增的算法/数据结构/知识点
- 在「更新日志」章节中新增今天的日期条目（格式：`### YYYY-MM-DD`）
- 详细列出今日的所有新增内容（代码文件、算法题、README章节等）
- 确保更新日志条目完整反映今日学习进度

**4. 向用户展示更新内容**
- 清晰展示对 `README.md` 做了哪些修改
- 展示完整的git状态（哪些文件会被提交）
- **明确询问用户**："README更新内容是否正确？是否可以继续执行git提交和推送？"
- **等待用户明确批准**（用户回复"是"、"可以"、"确认"、"批准"等）

### 阶段二：提交和推送（仅在用户批准后执行）

**5. git add**
- 添加所有修改和新增的文件到暂存区
- 不要添加 `.claude/settings.local.json`（配置文件，不应提交）

**6. git commit**
- 生成有意义的commit message，格式：
  ```
  YYYY-MM-DD：简短总结（20字以内）
  
  - 详细内容1
  - 详细内容2
  - 详细内容3
  ```
- commit message应该反映今日的核心学习内容

**7. git push**
- 推送到 `origin main`
- 确认推送成功

**8. 完成反馈**
- 告知用户提交和推送已完成
- 显示本次提交的SHA和消息

## 重要规则

1. **必须等待用户确认**：阶段一完成后，必须等待用户明确批准才能进入阶段二
2. **不要自动推送**：除非用户明确同意，否则不执行任何git commit或push操作
3. **README更新规范**：
   - 使用今天的实际日期（YYYY-MM-DD格式）
   - 更新日志应该详细、准确、完整
   - 保持README的格式一致性
4. **文件筛选**：不要提交 `.claude/settings.local.json`
5. **错误处理**：如果git命令失败，清晰报告错误，不要继续后续步骤
