## 二分查找基础

### 什么是二分查找？

二分查找（Binary Search）是一种高效的搜索算法。其核心思想是：**通过某种规律，每次将搜索范围缩小一半**，直到找到目标或确定不存在。

最经典的应用场景是在**有序数组**中查找特定元素：通过比较中间元素与目标值的大小关系，决定在左半部分还是右半部分继续查找。

### 算法特点

- **前提条件**：
  - **经典场景**：数组有序（升序或降序）
  - **本质要求**：能通过某种规律判断目标在哪一半，从而排除另一半
  - **重要理解**：二分的本质不是"有序性"，而是"可排除性"
- **时间复杂度**：O(log n) - 每次将搜索范围减半
- **空间复杂度**：O(1) - 只需要常数个额外变量
- **核心思想**：分治法，不断缩小搜索范围

**二分思想的应用场景**：
1. **有序数组查找**：利用大小关系排除一半（题目1、题目2）
2. **无序数组但有局部规律**：利用局部单调性排除一半（题目3：局部最小值）
3. **答案在某个范围内**：二分答案，判断可行性
4. **旋转数组、峰值查找** 等变体问题

### 为什么这么快？

对于长度为 n 的数组：
- 线性查找：最坏需要比较 n 次
- 二分查找：最坏需要比较 log₂n 次

**示例**：在 1000 个元素中查找
- 线性查找：最多 1000 次
- 二分查找：最多 10 次（2¹⁰ = 1024）

**示例**：在 1,000,000 个元素中查找
- 线性查找：最多 1,000,000 次
- 二分查找：最多 20 次（2²⁰ = 1,048,576）

差距非常明显！这就是为什么二分查找如此重要。

## 算法题

### 题目1：在有序数组中查找目标值

**问题描述**：在一个有序数组中，找一个数字是否存在，如果存在返回其索引，否则返回 -1。

**要求**：
- 时间复杂度：O(log n)
- 空间复杂度：O(1)

**示例**：
- 输入：`arr = [1, 3, 5, 7, 9, 11]`, `target = 7`，输出：`3`
- 输入：`arr = [1, 3, 5, 7, 9, 11]`, `target = 6`，输出：`-1`

**算法思路**：

1. 初始化左右边界：`left = 0`, `right = arr.length - 1`
2. 循环条件：`left <= right`（**注意必须是 <=**，因为当 left == right 时，仍有一个元素需要检查）
3. 计算中点：`middle = left + (right - left) / 2`（避免 `(left + right) / 2` 可能的整数溢出）
4. 比较中间元素：
   - 如果 `arr[middle] == target`，找到目标，返回 `middle`
   - 如果 `arr[middle] < target`，说明目标在右半部分，更新 `left = middle + 1`
   - 如果 `arr[middle] > target`，说明目标在左半部分，更新 `right = middle - 1`
5. 如果循环结束仍未找到，返回 -1

**核心代码**：

```java
public static int binarySearch(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;
    
    // while 循环的条件必须是 >=，因为即使是 = 时，所指向的元素也需要检查
    while(left <= right) {
        int middle = left + (right - left) / 2;
        if(arr[middle] == target) {
            return middle;
        } else if(arr[middle] < target) {
            left = middle + 1;
        } else {
            right = middle - 1;
        }
    }
    return -1;
}
```

**关键点解析**：

1. **为什么用 `left <= right` 而不是 `left < right`？**
   - 当 `left == right` 时，还有一个元素未检查
   - 如果用 `<`，最后一个元素会被遗漏

2. **为什么计算中点用 `left + (right - left) / 2`？**
   - 避免 `(left + right)` 可能超出整数最大值导致溢出
   - 数学上等价：`left + (right - left) / 2 = (2*left + right - left) / 2 = (left + right) / 2`

3. **为什么边界更新是 `middle + 1` 和 `middle - 1`？**
   - 因为已经确认 `arr[middle]` 不是目标值
   - 下一次搜索可以排除 `middle` 位置

**时间复杂度分析**：
- 每次循环搜索范围减半：n → n/2 → n/4 → ... → 1
- 需要循环 log₂n 次
- 时间复杂度：O(log n)

### 题目2：找大于等于某个数的最左侧位置

**问题描述**：在有序数组中，找大于等于某个数的最左侧位置。如果不存在这样的数，返回 -1。

**要求**：
- 时间复杂度：O(log n)
- 空间复杂度：O(1)

**示例**：
- 输入：`arr = [1, 2, 2, 2, 3, 5, 7]`, `target = 2`，输出：`1`（第一个 2 的位置）
- 输入：`arr = [1, 2, 3, 5, 7]`, `target = 4`，输出：`3`（第一个大于等于 4 的位置，即 5 的位置）
- 输入：`arr = [1, 2, 3, 5, 7]`, `target = 8`，输出：`-1`（不存在大于等于 8 的元素）

**算法思路**：

这道题是二分查找的变体，核心区别在于：
- **普通二分查找**：找到目标值就立即返回
- **最左侧位置查找**：找到目标值后，不能立即返回，需要继续向左查找，看左边是否还有符合条件的元素

**步骤**：
1. 初始化左右边界和结果变量：`left = 0`, `right = arr.length - 1`, `index = -1`
2. 循环条件：`left <= right`
3. 计算中点：`middle = left + (right - left) / 2`
4. 比较中间元素：
   - 如果 `arr[middle] >= target`：
     - 记录当前位置：`index = middle`（这是一个候选答案）
     - 继续向左查找：`right = middle - 1`（看左边是否还有更合适的）
   - 如果 `arr[middle] < target`：
     - 目标在右半部分：`left = middle + 1`
5. 循环结束后返回 `index`

**核心代码**：

```java
public static int findLeftmostPosition(int[] arr, int target) {
    // 这道题，在找到 target 元素时不能直接返回，
    // 因为你不知道左边还有没有更合适的答案，所以必须到二分结束
    int left = 0;
    int right = arr.length - 1;
    int index = -1;
    
    while(left <= right) {
        int middle = left + (right - left) / 2;
        if(arr[middle] >= target) {
            index = middle;        // 记录候选位置
            right = middle - 1;    // 继续向左查找
        } else {
            left = middle + 1;
        }
    }
    return index;
}
```

**关键点解析**：

1. **为什么需要 `index` 变量？**
   - 不能在找到符合条件的元素时立即返回
   - 需要记录"当前找到的最左侧位置"，继续向左搜索
   - 如果左边还有更小的符合条件的位置，会更新 `index`

2. **为什么 `arr[middle] >= target` 时要 `right = middle - 1`？**
   - 虽然 `middle` 位置符合条件，但左边可能还有更左的符合条件的位置
   - 继续在左半部分搜索，看能否找到更左的答案

3. **为什么初始化 `index = -1`？**
   - 如果整个数组中不存在 >= target 的元素，`index` 始终保持 -1
   - 作为"未找到"的标志返回

**算法演示**（以 `arr = [1, 2, 2, 2, 3, 5, 7]`, `target = 2` 为例）：

```
初始：left = 0, right = 6, index = -1

第1轮：middle = 3, arr[3] = 2 >= 2
     更新 index = 3, right = 2 (继续向左找)

第2轮：left = 0, right = 2
     middle = 1, arr[1] = 2 >= 2
     更新 index = 1, right = 0 (继续向左找)

第3轮：left = 0, right = 0
     middle = 0, arr[0] = 1 < 2
     更新 left = 1

第4轮：left = 1 > right = 0，循环结束
     返回 index = 1
```

**与普通二分查找的对比**：

| 特性 | 普通二分查找 | 最左侧位置查找 |
|------|------------|--------------|
| 找到目标时 | 立即返回 | 记录位置并继续向左搜索 |
| 返回值类型 | 目标值的索引 | 最左侧符合条件的索引 |
| 条件判断 | `==` | `>=` |
| 适用场景 | 查找是否存在 | 查找边界位置 |

**时间复杂度**：O(log n) - 与普通二分查找相同

### 题目3：局部最小值问题

**问题描述**：给定一个无序数组 arr，保证相邻的每两个数都不相等，要求找到任意一个局部最小值（极小值）的位置。

**局部最小值定义**：
- 如果 `arr[0] < arr[1]`，则 `arr[0]` 是局部最小值
- 如果 `arr[n-1] < arr[n-2]`，则 `arr[n-1]` 是局部最小值
- 如果 `arr[i-1] > arr[i] < arr[i+1]`，则 `arr[i]` 是局部最小值

**要求**：
- 时间复杂度：O(log n)（好于 O(n)）
- 空间复杂度：O(1)

**示例**：
- 输入：`arr = [6, 5, 3, 4, 6, 7, 8]`，输出：`2`（arr[2] = 3 是局部最小值）
- 输入：`arr = [9, 8, 7, 6, 5, 4, 3]`，输出：`6`（整个数组递减，最后一个元素是局部最小）
- 输入：`arr = [1, 2, 3, 4, 5, 6, 7]`，输出：`0`（整个数组递增，第一个元素是局部最小）

**核心思想**：

这道题的巧妙之处在于：
1. **不需要找全局最小值**，只需要找任意一个局部最小值
2. **数组无序**，但可以利用"相邻元素不相等"和"单调性"的特点
3. **二分思想的应用**：根据中间位置与相邻元素的关系，判断哪个方向必然存在局部最小值

**为什么无序数组也能用二分？**

虽然数组无序，但我们可以根据**局部的单调性**来判断：
- 如果 `arr[0] > arr[1]`，说明从左端向右递减，必然存在一个局部最小值
- 如果 `arr[n-1] > arr[n-2]`，说明从右端向左递减，必然存在一个局部最小值
- 如果两端都向内递减，中间必然存在"谷底"（局部最小值）

**算法思路**：

**步骤1：特殊情况处理**
- 空数组或长度为0：返回 -1
- 只有一个元素：返回 0（单个元素必然是局部最小值）

**步骤2：检查两端**
- 如果 `arr[0] < arr[1]`，则 `arr[0]` 是局部最小值，返回 0
- 如果 `arr[n-1] < arr[n-2]`，则 `arr[n-1]` 是局部最小值，返回 n-1

**步骤3：二分查找（搜索范围：`[1, n-2]`）**

排除两端后，进行二分查找：
1. 计算中点：`middle = left + (right - left) / 2`
2. 检查 `arr[middle]` 是否是局部最小值：
   - 如果 `arr[middle] < arr[middle-1]` 且 `arr[middle] < arr[middle+1]`，找到答案，返回 `middle`
3. 根据单调性决定搜索方向：
   - 如果 `arr[middle] > arr[middle-1]`：说明从 middle 向左递增，左侧必然存在局部最小值，更新 `right = middle - 1`
   - 否则（`arr[middle] < arr[middle-1]`）：说明从 middle 向右递减，右侧必然存在局部最小值，更新 `left = middle + 1`

**为什么这个策略能保证找到局部最小值？**

关键在于理解"必然存在"的逻辑：

**情况1：左边递增（`arr[middle] > arr[middle-1]`）**
```
... arr[left] ... arr[middle-1] < arr[middle] ...
```
- 从 `arr[left]` 到 `arr[middle]` 是递增的
- 但 `arr[0] > arr[1]`（否则步骤2已返回）
- 所以在 `[1, middle-1]` 区间内，必然存在一个"谷底"（局部最小值）

**情况2：右边递减（`arr[middle] < arr[middle-1]`）**
```
... arr[middle] > arr[middle+1] ... arr[right] ...
```
- 从 `arr[middle]` 到 `arr[right]` 是递减的
- 但 `arr[n-1] > arr[n-2]`（否则步骤2已返回）
- 所以在 `[middle+1, n-2]` 区间内，必然存在一个"谷底"（局部最小值）

**核心代码**：

```java
public static int findLocalMinimum(int[] arr) {
    // 最先对 arr 数组本身做合法性校验
    if(arr == null || arr.length == 0) {
        return -1;
    }
    if(arr.length == 1) {
        return 0;
    }
    
    // 再判断数组两端是否是局部最小值
    // 若不是的话【左侧递减，右侧递增】，则中间部分必然存在局部最小值
    int n = arr.length;
    if(arr[0] < arr[1]) {
        return 0;
    } else if(arr[n-1] < arr[n-2]) {
        return n - 1;
    }
    
    // 排除了两个端点之后，进入二分逻辑
    int left = 1;
    int right = n - 2;
    while(left <= right) {
        int middle = left + (right - left) / 2;
        if(arr[middle] < arr[middle-1] && arr[middle] < arr[middle+1]) {
            // 找到局部最小值
            return middle;
        } else if(arr[middle] > arr[middle-1]) {
            // 左边递增，说明左边范围内一定存在局部最小值
            // 由于采用【左闭右闭】的方式，middle 本身已经被证否，所以 middle-1
            right = middle - 1;
        } else {
            // 右边递减，说明右边范围内一定存在局部最小值
            left = middle + 1;
        }
    }
    
    return -1;  // 理论上不会执行到这里
}
```

**算法演示**（以 `arr = [6, 5, 3, 4, 6, 7, 8]` 为例）：

```
步骤1：检查特殊情况
- 长度 = 7，不是特殊情况

步骤2：检查两端
- arr[0] = 6 > arr[1] = 5，不是局部最小
- arr[6] = 8 > arr[5] = 7，不是局部最小
- 说明左端递减，右端递增，中间必有谷底

步骤3：二分查找
初始：left = 1, right = 5

第1轮：middle = 3
- arr[3] = 4
- arr[2] = 3 < arr[3]，不满足 arr[middle] < arr[middle-1]
- 说明左边递增，向左搜索
- 更新 right = 2

第2轮：left = 1, right = 2
- middle = 1
- arr[1] = 5
- arr[0] = 6 > arr[1]，arr[1] < arr[2] = 3？不满足
- arr[1] > arr[2]，说明右边递减，向右搜索
- 更新 left = 2

第3轮：left = 2, right = 2
- middle = 2
- arr[2] = 3
- arr[1] = 5 > arr[2] ✓, arr[2] < arr[3] = 4 ✓
- 找到局部最小值，返回 2
```

**关键点解析**：

1. **为什么搜索范围是 `[1, n-2]`？**
   - 因为步骤2已经检查了 `arr[0]` 和 `arr[n-1]`
   - 避免 `middle-1` 和 `middle+1` 越界

2. **为什么这个算法一定能找到答案？**
   - 数组两端的单调性保证了中间必然存在谷底
   - 每次二分都选择"下降"的方向，最终会收敛到谷底

3. **这道题为什么可以用二分？**
   - 关键不是"数组有序"，而是"能根据中间位置的信息排除一半的搜索空间"
   - 通过局部单调性判断，我们可以确定哪一半必然包含答案

4. **时间复杂度为什么是 O(log n)？**
   - 每次循环搜索范围减半
   - 最多需要 log₂n 次比较

**与传统二分查找的区别**：

| 特性 | 传统二分查找 | 局部最小值 |
|------|------------|-----------|
| 数组特性 | 必须有序 | 无序，但相邻不相等 |
| 查找目标 | 特定值 | 满足条件的位置 |
| 判断依据 | 与目标值比较 | 与相邻元素比较 |
| 核心思想 | 有序性 | 局部单调性 |
| 结果唯一性 | 唯一（如果存在） | 可能有多个 |

**这道题的启示**：

二分查找的本质不是"有序数组查找"，而是**通过某种规律，每次排除一半搜索空间**。只要能找到这个规律，就可以应用二分思想，即使数组本身无序。

**时间复杂度**：O(log n) - 每次搜索范围减半

**空间复杂度**：O(1) - 只使用常数个变量
