## 时间复杂度基本概念

### 什么是时间复杂度？

时间复杂度用来衡量算法执行时间随输入规模增长的变化趋势。我们使用大O表示法（Big O notation）来描述这种增长趋势。

### 时间复杂度的计算规则

#### 1. 去除低阶项，保留最高阶项

当算法的时间复杂度包含多项时，我们只关注增长最快的那一项。

例如：`T(n) = 3n² + 2n + 1`，时间复杂度为 `O(n²)`

- 当 n 足够大时，n² 的增长速度远超 2n 和 1
- 低阶项 2n 和常数项 1 对整体趋势影响很小，可以忽略

#### 2. 去除最高阶项的系数

在描述时间复杂度时，我们忽略常数系数。

例如：`T(n) = 5n²`，时间复杂度为 `O(n²)`，而不是 `O(5n²)`

- 系数不改变增长的数量级
- 大O表示法关注的是增长趋势，而非精确的执行时间

### 同阶复杂度的实际性能差异

需要特别注意的是：**相同时间复杂度的算法，实际运行时间可能差异很大**。

虽然理论上 `O(n)` 级别的算法增长趋势相同，但不同的运算操作实际耗时不同：

- **位运算**（如 `&`, `|`, `^`, `<<`, `>>`）通常是最快的
- **加减法**比乘除法更快
- **乘法**比除法更快
- **整数运算**比浮点数运算更快
- **数组访问**比链表遍历更快

#### 示例对比：系数不能说明一切

假设有两个同为 `O(n)` 时间复杂度的算法：

**算法A：** 使用位运算，执行10次循环（系数为10）
```java
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += (i & 1);      // 位运算
    sum += (i >> 1);     // 位运算
    sum += (i << 1);     // 位运算
    sum += (i ^ 1);      // 位运算
    sum += (i | 1);      // 位运算
    sum += (i & 2);      // 位运算
    sum += (i >> 2);     // 位运算
    sum += (i << 2);     // 位运算
    sum += (i ^ 2);      // 位运算
    sum += (i | 2);      // 位运算
}
```

**算法B：** 使用算术运算，执行3次循环（系数为3）
```java
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += i % 10;       // 取模运算
    sum += i / 10;       // 除法运算
    sum += i * i;        // 乘法运算
}
```

**理论分析：**
- 算法A：`T(n) = 10n`，时间复杂度 `O(n)`
- 算法B：`T(n) = 3n`，时间复杂度 `O(n)`

**实际结果：** 尽管算法A的系数（10）远大于算法B的系数（3），但由于位运算的执行速度远快于算术运算（尤其是取模和除法），**算法A在实际运行中反而更快**。

这个例子说明：**同阶复杂度下，不能只看系数大小来判断性能，必须考虑具体的运算类型**。

### 实际测试的重要性

因此，在对比同阶复杂度的算法时，**不能单纯依赖理论分析，需要实际运行程序并测量执行时间**。

影响实际性能的因素包括：
- 具体的运算类型
- 内存访问模式
- 缓存命中率
- 编译器优化
- 硬件特性

**结论：** 时间复杂度是算法性能的重要指标，但在选择具体实现时，还需要通过实际测试来验证性能表现。

## 空间复杂度基本概念

### 什么是空间复杂度？

空间复杂度用来衡量算法在执行过程中所需额外存储空间随输入规模增长的变化趋势。同样使用大O表示法来描述。

### 空间复杂度的组成

算法使用的总空间包括：

1. **输入空间**：存储输入数据所需的空间
2. **辅助空间**：算法执行过程中临时占用的额外空间（这是我们主要关注的）
3. **输出空间**：存储输出结果所需的空间

**注意：** 通常所说的空间复杂度指的是**辅助空间**，即算法本身需要的额外空间，不包括输入和输出占用的空间。

### 空间复杂度的计算

空间复杂度的计算规则与时间复杂度类似：
- 去除低阶项，保留最高阶项
- 去除常数系数
- 关注空间使用量的增长趋势

### 常见空间复杂度示例

#### O(1) - 常数空间

算法只使用固定数量的变量，不随输入规模变化。

```java
public int sum(int[] arr) {
    int total = 0;  // 只使用一个额外变量
    for (int num : arr) {
        total += num;
    }
    return total;
}
```

#### O(n) - 线性空间

算法需要的额外空间与输入规模成正比。

```java
public int[] copyArray(int[] arr) {
    int n = arr.length;
    int[] newArr = new int[n];  // 需要 n 个额外空间
    for (int i = 0; i < n; i++) {
        newArr[i] = arr[i];
    }
    return newArr;
}
```

#### O(log n) - 对数空间

常见于递归深度为 log n 的算法，如二分查找的递归实现。

```java
public int binarySearch(int[] arr, int target, int left, int right) {
    if (left > right) {
        return -1;
    }
    
    int mid = left + (right - left) / 2;
    
    if (arr[mid] == target) {
        return mid;
    } else if (arr[mid] < target) {
        return binarySearch(arr, target, mid + 1, right);  // 递归调用，栈空间 O(log n)
    } else {
        return binarySearch(arr, target, left, mid - 1);
    }
}
```

每次递归调用会在调用栈中占用空间，二分查找最多递归 log n 层。

#### O(n²) - 平方空间

需要创建二维数组或矩阵。

```java
public int[][] createMatrix(int n) {
    int[][] matrix = new int[n][n];  // 需要 n×n 个空间
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            matrix[i][j] = i * j;
        }
    }
    return matrix;
}
```

### 时间与空间的权衡

在算法设计中，时间复杂度和空间复杂度往往需要权衡：

- **用空间换时间**：通过额外的存储空间（如哈希表、缓存）来减少计算时间
- **用时间换空间**：减少空间占用，但可能需要重复计算

#### 示例：斐波那契数列

**方法1：递归实现**
```java
// 时间：O(2^n)，空间：O(n)（递归栈深度）
public int fibonacci(int n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}
```

**方法2：动态规划（数组）**
```java
// 时间：O(n)，空间：O(n)（需要数组存储）
public int fibonacciDP(int n) {
    if (n <= 1) return n;
    int[] dp = new int[n + 1];
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}
```

**方法3：动态规划（变量优化）**
```java
// 时间：O(n)，空间：O(1)（只用两个变量）
public int fibonacciOptimized(int n) {
    if (n <= 1) return n;
    int prev = 0, curr = 1;
    for (int i = 2; i <= n; i++) {
        int next = prev + curr;
        prev = curr;
        curr = next;
    }
    return curr;
}
```

这个例子展示了如何通过优化空间使用（从 O(n) 降到 O(1)）而保持相同的时间复杂度。

### 总结

- **时间复杂度**关注算法的执行速度
- **空间复杂度**关注算法的内存占用
- 两者往往需要权衡，没有绝对的最优解
- 实际应用中要根据具体场景选择：内存受限时优先考虑空间，性能要求高时优先考虑时间
